Methodology · Tool-role separation

Every surface has one job.

Mixing eval, production, and sensitive work on one endpoint is how stacks rot. Roles stay explicit.

Cursor
Daily frontier models · Agent · MCP (Obsidian, Docker gateway)
Ollama
Production open models on docker-host — not on the Mac
LM Studio
Mac-only eval + offline travel — never wired to Pi or prod loops
Pi / ollmcp
Sensitive CLI and vault chat on homelab models via Teleport/LAN
ai-api-proxy
Unified OpenAI-compatible routing with ollama/ prefixes

Methodology · Trust boundaries

Security is operationalized.

“Please don’t push” is not a control. Hooks, staging folders, and human-only remotes are.

Git

Cloud never pushes

  • Pre-push hooks on agent VMs
  • GitLab origin = system of record
  • GitHub mirror for Cloud reachability
  • Desktop human promotes and pushes
Vault

Write classes

  • Slack / Cloud → 0-inbox/ only
  • Desktop → PARA promotion authority
  • PII / credential hygiene in rules
  • MCP tools with path + extension guards

Methodology · Operating loop

Phases and gates — not vibe thrash.

Overloaded multi-goal prompts burn context. Explicit phases keep agents accurate and reviewable.

Turn design

One goal or named phases

  • Baseline before optimize
  • “Confirm when done” gates
  • Do-not-edit locks on authorship surfaces
  • Ask sync vs dual-commit before acting
Closeout

Durable session memory

  • MCP-16 session JSON under 4-resources/json/
  • Daily note completed tasks + summary
  • Tracker updates for active projects
  • Human push when ready

Memory architecture

The model is not the archive.

kbase_professional (PARA vault) + kbase_dev (monorepo) hold continuity. Agents read trackers via Obsidian MCP; RAG/pgvector deepens retrieval when needed.

Git SoR
